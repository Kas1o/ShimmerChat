
## ShimmerChat 2 下一阶段执行计划（可直接实施）

> 目标：先把“隐性错误难定位”问题解决，再补关键缺口与测试，保证后续开发可持续。

### 阶段 P0：稳定性与可观测性（最高优先级）

#### P0-1 统一错误处理（禁止 silent fail）（已经完成）
- 处理范围：Node。
- 实施动作：
  1. 定义统一执行结果结构（Success / Code / Message / Details）。
  2. 将“直接 return / 吞异常”的路径改为显式失败结果。
  3. UI 统一显示失败信息（至少包含 Message，可展开 Details）。
- 验收标准：
  - 任意 preset 缺失、tool 解析失败、KVData 不可用时，界面可见可读错误。
  - 日志可看到节点名、preset 名、agent/chat 标识。

---

### 阶段 P1：关键能力缺口修复

#### P1-1 ToolPresetNode 类型解析缓存                           (已经完成，统一工具扫描服务)
- 问题：当前每次解析都扫描程序集并实例化候选类型，开销过高。
- 实施动作：
  1. 增加静态缓存字典（toolName -> Type）。
  2. 首次扫描后复用缓存，支持线程安全读取。
  3. 记录首次构建与缓存命中日志。
- 验收标准：
  - 重复调用 ToolPresetNode 时不再重复全量扫描。
  - 相关功能行为与现有逻辑一致。

#### P1-2 SubAgentNode 输出模式补齐                            （已经完成、修复一些Bug）
- 实施动作：
  1. 保持 LastMessage 现有行为。
  2. 补充 FullJson 输出。
  3. 补充 None 输出（仅副作用，不回传文本）。
- 验收标准：
  - 三种输出模式可切换且行为可预测。
  - 旧流程迁移后不依赖临时兼容代码。

### 阶段 P2：编辑体验与预设可用性

#### P2-1 GenerationManager 树编辑体验补齐
- 实施动作：

  2. 先实现“上移/下移”重排（后续再考虑拖拽）。
  3. 支持跨树复制/粘贴（基于节点 JSON）。
- 验收标准：
  - 常见编辑动作无需手工改 JSON。
  - 树操作失败时有明确提示。

#### P2-2 Tool Manager 默认预设
- 实施动作：
  1. 提供可直接使用的默认工具预设。
  2. 默认生成环境修改器预设流程与 APISelect 节点交互方式对齐。
- 验收标准：
  - 新建 Agent 后可立即使用默认预设跑通基础流程。

## 建议实施顺序（严格执行）
1. P0-1 → P0-2（先解决可观测性与错误透明度）
2. P1-1 → P1-2 → P1-3（再补性能与功能缺口）

## Definition of Done（阶段完成标准）
- 代码：对应改动已合入并可运行。
- 验证：每个子任务有可复现的验证步骤与结果。
- 回归：关键路径测试通过。
- 可观测：失败时能在 UI/日志中定位具体节点与原因。
