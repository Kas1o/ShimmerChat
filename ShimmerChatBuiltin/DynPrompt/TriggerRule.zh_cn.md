# 触发规则（TriggerRule）

`TriggerRule` 属性定义了动态提示被激活的条件。
它本质上是一个由正则表达式模式和逻辑运算符（`|`、`&`、`!`、`(`、`)`）组成的表达式，根据输入文本评估为真或假。

## 语法
- **正则表达式模式**：这些是标准的正则表达式，用于匹配特定的文本模式。
- **逻辑运算符**：
  - `|`（或）：至少一个条件必须为真。
  - `&`（与）：所有条件必须为真。
  - `!`（非）：条件必须为假。
  - `(` 和 `)`：用于分组条件并控制评估顺序。

## 示例

```
"error" | "failure"
```
当输入文本包含 "error" 或 "failure" 时触发。

```
"success" & !"warning"
```
当输入文本包含 "success" 且不包含 "warning" 时触发。

```
("start" | "begin") & ("end" | "finish")
```
当输入文本包含 "start" 或 "begin"，并且同时包含 "end" 或 "finish" 时触发。

```
"[0-9]" & !"[a-zA-Z]"
```
当输入文本包含至少一个数字且不包含任何字母时触发。

## 转义特殊字符
如前所述，这些正则表达式模式用双引号括起来。

要在正则表达式模式中包含特殊字符，可能需要使用反斜杠（`\`）进行转义。例如，要匹配字面点（`.`），您需要在模式中使用 `"\\."`。第一个反斜杠用于字符串中的字面转义，第二个反斜杠用于正则表达式转义。

在实现中，我们使用 C# Newtonsoft.Json 库将这些正则表达式解析为字符串字面量，因此请确保同时遵循 JSON 字符串的转义规则。
